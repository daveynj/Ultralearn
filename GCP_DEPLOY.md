# Deploying UltraLearn to Google Cloud Platform

This guide will help you deploy the UltraLearn application to Google Cloud Run with a PostgreSQL database on Cloud SQL.

## Prerequisites

1.  A Google Cloud Platform (GCP) Project with billing enabled.
2.  Access to [Google Cloud Shell](https://shell.cloud.google.com) (recommended) or a local terminal with `gcloud` CLI installed and authenticated.

## Step 1: Initial Infrastructure Setup

We have provided a script to automate the creation of necessary resources (Cloud SQL, Artifact Registry, Secret Manager, Service Accounts).

1.  Open Google Cloud Shell.
2.  Clone your repository (if not already there) or upload the `scripts/` folder.
3.  Navigate to the project root.
4.  Run the setup script:

```bash
chmod +x scripts/setup-gcp.sh
./scripts/setup-gcp.sh
```

5.  Follow the interactive prompts:
    -   **Project ID**: Enter your GCP Project ID.
    -   **Region**: Press Enter for default (`us-central1`) or choose another.
    -   **Database Password**: Create a strong password for your database.
    -   **Secrets**: You will be prompted to enter values for API keys (OpenAI, Stripe, etc.). You can enter them now or press Enter to skip and fill them later in the GCP Console (Secret Manager).

The script will generate a file named `gcp-sa-key.json` and display the values you need for GitHub Secrets.

## Step 2: Configure GitHub Secrets

1.  Go to your GitHub repository.
2.  Navigate to **Settings** > **Secrets and variables** > **Actions**.
3.  Click **New repository secret**.
4.  Add the following secrets (values will be provided by the script output):

    -   **Name**: `GCP_PROJECT_ID`
        -   **Value**: Your Google Cloud Project ID.

    -   **Name**: `GCP_SA_KEY`
        -   **Value**: The entire content of the `gcp-sa-key.json` file generated by the script.

    -   **Name**: `GCP_SQL_INSTANCE_NAME`
        -   **Value**: The Cloud SQL connection name (e.g., `project:region:instance`).

    -   **Name**: `GCP_SA_EMAIL`
        -   **Value**: The email of the Runtime Service Account (e.g., `ultralearn-runtime@...`).

    > **Security Note:** Delete the `gcp-sa-key.json` file from your local machine or Cloud Shell after adding it to GitHub.

## Step 3: Deploy

Simply push your changes to the `main` branch:

```bash
git add .
git commit -m "Configure GCP deployment"
git push origin main
```

Go to the **Actions** tab in your GitHub repository to watch the deployment progress.
The workflow will:
1.  Build the Docker image.
2.  Push it to Google Artifact Registry.
3.  Run database migrations (using a Cloud Run Job).
4.  Deploy the application to Cloud Run.

## Step 4: Post-Deployment Configuration

Once the deployment is successful, Cloud Run will provide a public URL for your application (e.g., `https://ultralearn-xyz-uc.a.run.app`).

1.  Copy this URL.
2.  Go to **Google Cloud Console** > **Security** > **Secret Manager**.
3.  Update the following secrets with the new URL:
    -   `NEXTAUTH_URL`: Set this to your Cloud Run URL (no trailing slash).
    -   `NEXT_PUBLIC_APP_URL`: Set this to your Cloud Run URL (no trailing slash).
    -   **Note:** If you are using Google OAuth, add this URL (plus `/api/auth/callback/google`) to your **Authorized redirect URIs** in the Google Cloud Console (APIs & Services > Credentials).
    -   **Note:** If you are using Stripe, update your Stripe Webhook endpoint with this URL (plus `/api/webhooks/stripe`).
4.  To apply these changes, you can either:
    -   Trigger a new deployment (push a commit).
    -   Or go to Cloud Run > `ultralearn` > **Edit & Deploy new revision** > **Deploy**.

## Troubleshooting

-   **Database Connection Failed**: Ensure the `DATABASE_URL` secret in Secret Manager matches the format output by the setup script. It should look like `postgresql://postgres:PASSWORD@localhost/ultralearn?host=/cloudsql/PROJECT:REGION:INSTANCE`. Also ensure `GCP_SQL_INSTANCE_NAME` is correct in GitHub Secrets.
-   **Migration Failed**: Check the Cloud Run Jobs logs for `ultralearn-migrate`. Ensure the database credentials are correct and the instance is running.
-   **500 Errors**: Check the Cloud Run logs. Ensure all required environment variables (API keys) are set in Secret Manager.
